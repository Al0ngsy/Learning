# Agile Philosophy

## Agile Principles

### Agile and Its Characteristics

Agile is an iterative approach to project management focused on delivering value quickly through small increments and customer feedback.

Key characteristics include:

- Adaptive planning.
- Evolutionary development.
- Early delivery.
- Continuous improvement.
- Responsiveness to change.

### The Agile Manifesto Values

- Individuals and interactions over processes and tools.
- Working software over comprehensive documentation.
- Customer collaboration over contract negotiation.
- Responding to change over strictly following a plan.

### Agile Software Development Practices

Agile development uses small, co-located, cross-functional, self-organizing teams.

It focuses on building what is needed based on customer feedback rather than rigidly following initial plans. It also encourages flexibility, transparency, and collaboration within the team and with customers.

## Methodologies Overview

### Waterfall Development Approach

Waterfall is a linear, phase-based process starting with requirements gathering, followed by design, coding, integration, testing, and deployment, with strict entrance and exit criteria for each phase.

Common limitations include:

- Limited flexibility for changes.
- Delayed feedback until late phases.
- Costly fixes for late-discovered issues.
- Long lead times.
- Siloed teamwork and weaker communication.

### Extreme Programming (XP) Approach

XP is an iterative, Agile-inspired methodology emphasizing short feedback loops at multiple levels, from major releases to daily stand-ups and pair programming.

XP values include:

- Simplicity (avoiding over-engineering).
- Communication among team members.
- Continuous feedback.
- Mutual respect.
- Courage to be honest about estimates and challenges.

### Kanban Methodology

Originating from Japanese manufacturing, Kanban focuses on visualizing workflow using boards, limiting work in progress to avoid bottlenecks, and managing flow for continuous improvement.

Core principles include:

- Making process policies explicit.
- Understanding the definition of done.
- Fostering continuous feedback and process enhancement.

## Working Agile

### Working in Small Batches

Small batches reduce waste by enabling faster feedback and quicker detection of errors compared to large batches.

Single-piece flow allows inspection of the first finished product in seconds, enabling early problem identification and adjustment.

### Minimum Viable Product (MVP)

An MVP is the smallest product version that tests a hypothesis and enables learning, not just the first phase or beta release.

Iterative MVPs help teams pivot or persevere based on customer feedback, leading to a product closer to customer needs.

### Behavior Driven Development (BDD) and Test Driven Development (TDD)

BDD tests the system from the outside in, focusing on user behavior and using a common syntax (Given-When-Then) understandable by developers and stakeholders.

TDD tests the system from the inside out at the unit level, writing tests before code to ensure correctness and enable safe refactoring.

### Pair Programming

Two programmers work together, one writing code and the other reviewing in real time, which improves code quality and catches defects early.

This also supports mentorship and knowledge sharing, making debugging and maintenance cheaper in the long run.
